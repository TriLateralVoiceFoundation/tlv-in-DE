## Code of Conduct

By participating in this project, you agree to uphold our values of **dignity, authenticity, and accessibility** in all interactions.

We are committed to:
- Respectful, inclusive communication
- Acknowledging diverse perspectives and lived experiences
- Creating a safe space for authentic contribution
- Supporting contributors at all experience levels

**Unacceptable behavior** includes harassment, discrimination, personal attacks, or any conduct that damages the dignity of community members.

To report violations, contact: conduct@tlvfoundation.org

## How Can I Contribute?

## Reporting Issues

Found a problem? We want to know!

**Before submitting an issue:**
1. Check [existing issues](https://gitlab.com/trilateral-voice-tlv-foundation/tlv-us-eng/-/issues) to avoid duplicates
2. Review the relevant protocol documentation to ensure it's not a misunderstanding

**When reporting an issue:**
1. Go to [GitLab Issues](https://gitlab.com/trilateral-voice-tlv-foundation/tlv-us-eng/-/issues)
2. Click "New Issue"
3. Use a clear, descriptive title
4. Provide details:
   - Which document/protocol is affected?
   - What is the issue? (error, unclear language, missing information?)
   - What did you expect?
   - How can it be improved?
5. Add relevant labels (e.g., "documentation," "protocol," "bug")

**Examples of good issue titles:**
- "Transcription Protocol: Unclear baseline establishment process"
- "Emotional Vocabulary: Missing entry for 'overwhelmed'"
- "README: Broken link to White Paper"

## Suggesting Enhancements

Have an idea to improve TLV? We'd love to hear it!

**Enhancement suggestions might include:**
- New sections or features in protocols
- Additional examples or case studies
- Better organization or navigation
- Accessibility improvements
- New protocol applications

**To suggest an enhancement:**
1. Open a [New Issue](https://gitlab.com/trilateral-voice-tlv-foundation/tlv-us-eng/-/issues)
2. Use label "enhancement"
3. Describe:
   - What you'd like to see added/changed
   - Why this would be valuable
   - How it aligns with TLV's mission and principles
   - Any examples or resources that support your suggestion

**Note**: Enhancements should align with TLV's core mission of documenting and honoring the full dimensionality of human communication.

## Contributing Documentation

We welcome improvements to existing documentation and new documentation that helps people understand and use TLV.

**What you can contribute:**
- Clarifications to existing protocols
- Expanded explanations
- Additional examples
- Tutorials or how-to guides
- Visual aids (diagrams, flowcharts)
- Glossaries or quick references

**Process:**
1. Fork the repository
2. Create a new branch for your changes (`git checkout -b improve-transcription-protocol`)
3. Make your changes to the relevant `.md` files
4. Ensure your changes follow the existing style and structure
5. Submit a Merge Request (see below)

**Documentation style guidelines:**
- Use clear, accessible language
- Define technical terms when first used
- Include concrete examples
- Maintain the three-voice framework perspective
- Follow existing formatting conventions
- Respect the CC BY-NC-SA 4.0 license

## Translation Contributions

TLV aims to be globally accessible. Translation contributions are highly valued!

**Translation priorities:**
1. Core protocols (Communication, Transcription, Translation)
2. White Paper and Blue Paper
3. Common Speech Intro
4. Emotional Vocabulary

**Translation guidelines:**
- Maintain dimensional integrity—don't flatten content
- Adapt cultural context appropriately
- Preserve the three-voice structure
- Include translator notes where cultural adaptation is significant
- Follow TLV Translation Protocol methodology

**To contribute a translation:**
1. Contact: translations@tlvfoundation.org
2. Indicate which document(s) you'd like to translate
3. Specify target language and cultural context
4. Coordinate with the TLV team to ensure quality and consistency

**Translation will be credited** and licensed under the same CC BY-NC-SA 4.0 terms.

## Sharing Use Cases

Real-world applications strengthen TLV and help others understand its value.

**You can share:**
- How you've used TLV in your practice
- Insights from applying protocols
- Challenges you've encountered and how you addressed them
- Anonymized examples of TLV in action

## Contribution Guidelines

### Merge Request Process

1. **Fork the repository**
2. **Create a feature branch** from `main`
   ```bash
   git checkout -b feature/your-contribution-name
   ```
3. **Make your changes**
   - Follow existing formatting
   - Include clear commit messages
   - Test that links work and markdown renders correctly
4. **Commit your changes**
   ```bash
   git commit -m "Brief description of changes"
   ```
5. **Push to your fork**
   ```bash
   git push origin feature/your-contribution-name
   ```
6. **Create a Merge Request**
   - Go to the TLV GitLab repository
   - Click "New Merge Request"
   - Select your branch
   - Provide clear description of changes
   - Reference any related issues

### Commit Message Guidelines

Write clear, descriptive commit messages:

**Good examples:**
- `Add example to Transcription Protocol section 3.2`
- `Fix broken link in README to White Paper`
- `Clarify baseline establishment process`
- `Expand emotional vocabulary entry for 'anxious'`

**Avoid:**
- `Update file`
- `Fixes`
- `Changes`

### File Naming Conventions

Follow existing conventions:
- Use hyphens for spaces
- Include version number in protocol files: `(RC0001)`
- Include language version: `(US-English)`
- Use descriptive names

**Example**: `TriLateral-Voice (TLV) Protocol-Name (RC0001)(US-English).md`

## Review Process

1. **Submission**: You submit a Merge Request or Issue
2. **Initial Review**: TLV team reviews within 10-15 business days
3. **Discussion**: Team may ask questions or request modifications
4. **Revision**: You make any requested changes
5. **Approval**: Once approved, changes are merged
6. **Credit**: Contributors are acknowledged

**Timeline**: Most contributions are reviewed within 2 weeks. Complex contributions (new protocols, major changes) may take longer.
