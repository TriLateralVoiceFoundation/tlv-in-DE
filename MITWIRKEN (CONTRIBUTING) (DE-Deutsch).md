# Beitrag zu TriLateral Voice (TLV)

**Übersetzungshinweis:** Dieses Dokument wurde gemäß dem TLV-Übersetzungsprotokoll (RC0001) vom amerikanischen Englisch ins Deutsche übersetzt. Diese Übersetzung zielt darauf ab, die vollständige Bedeutung, den emotionalen Kontext und die kulturelle Integrität des Originaldokuments zu bewahren, und dient am besten als Ressource zum Verständnis des ursprünglichen amerikanisch-englischen Inhalts, da bestimmte Konzepte im Deutschen möglicherweise nicht existieren oder anders verwendet werden.

---

Vielen Dank für Ihr Interesse, zum TriLateral Voice (TLV) Framework beizutragen! Wir freuen uns darauf, diese Gemeinschaft gemeinsam aufzubauen.

Dies ist eine frühe Veröffentlichung (RC0001), und wir begrüßen aktiv Beiträge von Praktikern, Forschern, Pädagogen, Übersetzern und allen, die sich für authentische menschliche Kommunikation begeistern.

## Wie Kann Ich Beitragen?

Es gibt viele Möglichkeiten, zu TLV beizutragen:

### 1. **Rückmeldung zu Protokollen Geben**
- Protokolle in Ihrem Kontext testen (persönlich, beruflich, Forschung)
- Teilen Sie, was gut funktioniert und was verbessert werden könnte
- Klarstellungen oder Ergänzungen vorschlagen

### 2. **Probleme Melden**
- Dokumentationsfehler oder Tippfehler
- Unklare Anweisungen
- Fehlende Informationen
- Technische Probleme mit Dateien

### 3. **Verbesserungen Vorschlagen**
- Neue Protokollfunktionen
- Zusätzliche Anwendungen oder Anwendungsfälle
- Verbesserte Zugänglichkeit
- Bessere Organisation oder Struktur

### 4. **Dokumentation Beitragen**
- Erklärungen erweitern
- Beispiele hinzufügen
- Tutorials oder Anleitungen erstellen
- Klarheit verbessern

### 5. **Materialien Übersetzen**
- Protokolle in andere Sprachen übersetzen
- Emotionales Vokabular an kulturelle Kontexte anpassen
- Helfen Sie, TLV global zugänglich zu machen

### 6. **Anwendungsfälle Teilen**
- Dokumentieren Sie, wie Sie TLV verwenden
- Teilen Sie reale Anwendungen
- Anonymisierte Beispiele beitragen
- Fallstudien schreiben

### 7. **An Diskussionen Teilnehmen**
- Nehmen Sie an Gemeinschaftsdiskussionen in GitLab Issues teil
- Teilen Sie Erkenntnisse aus Ihrer Praxis
- Helfen Sie anderen, TLV zu verstehen
- Verbindungen mit anderen Praktikern aufbauen

## Probleme Melden

Ein Problem gefunden? Wir möchten es wissen!

**Bevor Sie ein Problem einreichen:**
1. Überprüfen Sie [bestehende Probleme](https://gitlab.com/trilateral-voice-tlv-foundation/tlv-us-eng/-/issues), um Duplikate zu vermeiden
2. Überprüfen Sie die relevante Protokolldokumentation, um sicherzustellen, dass es sich nicht um ein Missverständnis handelt

**Beim Melden eines Problems:**
1. Gehen Sie zu [GitLab Issues](https://gitlab.com/trilateral-voice-tlv-foundation/tlv-us-eng/-/issues)
2. Klicken Sie auf "New Issue"
3. Verwenden Sie einen klaren, beschreibenden Titel
4. Geben Sie Details an:
   - Welches Dokument/Protokoll ist betroffen?
   - Was ist das Problem? (Fehler, unklare Sprache, fehlende Informationen?)
   - Was haben Sie erwartet?
   - Wie kann es verbessert werden?
5. Fügen Sie relevante Labels hinzu (z.B. "documentation", "protocol", "bug")

**Beispiele für gute Problemtitel:**
- "Transkriptionsprotokoll: Unklarer Prozess zur Festlegung der Basislinie"
- "Emotionales Vokabular: Fehlender Eintrag für 'overwhelmed'"
- "README: Defekter Link zum White Paper"

## Verbesserungen Vorschlagen

Haben Sie eine Idee zur Verbesserung von TLV? Wir würden sie gerne hören!

**Verbesserungsvorschläge können umfassen:**
- Neue Abschnitte oder Funktionen in Protokollen
- Zusätzliche Beispiele oder Fallstudien
- Bessere Organisation oder Navigation
- Zugänglichkeitsverbesserungen
- Neue Protokollanwendungen

**Um eine Verbesserung vorzuschlagen:**
1. Öffnen Sie ein [Neues Problem](https://gitlab.com/trilateral-voice-tlv-foundation/tlv-us-eng/-/issues)
2. Verwenden Sie das Label "enhancement"
3. Beschreiben Sie:
   - Was Sie hinzugefügt/geändert sehen möchten
   - Warum dies wertvoll wäre
   - Wie es mit TLVs Mission und Prinzipien übereinstimmt
   - Alle Beispiele oder Ressourcen, die Ihren Vorschlag unterstützen

**Hinweis**: Verbesserungen sollten mit TLVs zentraler Mission übereinstimmen, die volle Dimensionalität der menschlichen Kommunikation zu dokumentieren und zu ehren.

## Dokumentation Beitragen

Wir begrüßen Verbesserungen an bestehender Dokumentation und neue Dokumentation, die Menschen hilft, TLV zu verstehen und zu verwenden.

**Was Sie beitragen können:**
- Klarstellungen zu bestehenden Protokollen
- Erweiterte Erklärungen
- Zusätzliche Beispiele
- Tutorials oder Schritt-für-Schritt-Anleitungen
- Visuelle Hilfsmittel (Diagramme, Flussdiagramme)
- Glossare oder Schnellreferenzen

**Prozess:**
1. Forken Sie das Repository
2. Erstellen Sie einen neuen Branch für Ihre Änderungen (`git checkout -b improve-transcription-protocol`)
3. Nehmen Sie Ihre Änderungen an den relevanten `.md` Dateien vor
4. Stellen Sie sicher, dass Ihre Änderungen dem bestehenden Stil und der Struktur folgen
5. Reichen Sie einen Merge Request ein (siehe unten)

**Dokumentationsstil-Richtlinien:**
- Verwenden Sie klare, zugängliche Sprache
- Definieren Sie technische Begriffe bei der ersten Verwendung
- Fügen Sie konkrete Beispiele ein
- Wahren Sie die Perspektive des Drei-Stimmen-Frameworks
- Befolgen Sie bestehende Formatierungskonventionen
- Respektieren Sie die CC BY-NC-SA 4.0 Lizenz

## Übersetzungsbeiträge

TLV zielt darauf ab, global zugänglich zu sein. Übersetzungsbeiträge werden sehr geschätzt!

**Übersetzungsprioritäten:**
1. Kernprotokolle (Kommunikation, Transkription, Übersetzung)
2. White Paper und Blue Paper
3. Allgemeinsprachliche Einführung
4. Emotionales Vokabular

**Übersetzungsrichtlinien:**
- Wahren Sie die dimensionale Integrität—flachen Sie den Inhalt nicht ab
- Passen Sie den kulturellen Kontext angemessen an
- Bewahren Sie die Drei-Stimmen-Struktur
- Fügen Sie Übersetzernotizen hinzu, wo kulturelle Anpassung bedeutsam ist
- Folgen Sie der TLV-Übersetzungsprotokoll-Methodologie

**Um eine Übersetzung beizutragen:**
1. Kontaktieren Sie: translations@tlvfoundation.org
2. Geben Sie an, welche(s) Dokument(e) Sie übersetzen möchten
3. Geben Sie Zielsprache und kulturellen Kontext an
4. Koordinieren Sie mit dem TLV-Team, um Qualität und Konsistenz sicherzustellen

**Übersetzungen werden anerkannt** und unter denselben CC BY-NC-SA 4.0 Bedingungen lizenziert.

## Anwendungsfälle Teilen

Reale Anwendungen stärken TLV und helfen anderen, seinen Wert zu verstehen.

**Sie können teilen:**
- Wie Sie TLV in Ihrer Praxis verwendet haben
- Erkenntnisse aus der Anwendung von Protokollen
- Herausforderungen, denen Sie begegnet sind, und wie Sie sie angegangen sind
- Anonymisierte Beispiele von TLV in Aktion
- Forschungsergebnisse unter Verwendung der TLV-Methodologie

**Um einen Anwendungsfall zu teilen:**
1. Öffnen Sie ein Problem mit dem Label "use-case" ODER
2. E-Mail: usecases@tlvfoundation.org

**Einschließen:**
- Kontext (persönlich, beruflich, Forschung usw.)
- Welche(s) Protokoll(e) Sie verwendet haben
- Was gut funktioniert hat
- Was verbessert werden könnte
- Erkenntnisse oder gelernte Lektionen
- Erlaubnis zum Teilen (anonymisiert falls erforderlich)

**Datenschutzhinweis**: Anonymisieren Sie immer identifizierende Informationen und holen Sie Zustimmung ein, bevor Sie Beispiele teilen, die andere betreffen.

## Beitragsrichtlinien

### Merge Request Prozess

1. **Forken Sie das Repository**
2. **Erstellen Sie einen Feature-Branch** von `main`
   ```bash
   git checkout -b feature/your-contribution-name
   ```
3. **Nehmen Sie Ihre Änderungen vor**
   - Folgen Sie der bestehenden Formatierung
   - Fügen Sie klare Commit-Nachrichten ein
   - Testen Sie, dass Links funktionieren und Markdown korrekt gerendert wird
4. **Committen Sie Ihre Änderungen**
   ```bash
   git commit -m "Kurze Beschreibung der Änderungen"
   ```
5. **Pushen Sie zu Ihrem Fork**
   ```bash
   git push origin feature/your-contribution-name
   ```
6. **Erstellen Sie einen Merge Request**
   - Gehen Sie zum TLV GitLab Repository
   - Klicken Sie auf "New Merge Request"
   - Wählen Sie Ihren Branch
   - Geben Sie eine klare Beschreibung der Änderungen
   - Verweisen Sie auf alle zugehörigen Probleme

### Richtlinien für Commit-Nachrichten

Schreiben Sie klare, beschreibende Commit-Nachrichten:

**Gute Beispiele:**
- `Beispiel zu Transkriptionsprotokoll Abschnitt 3.2 hinzufügen`
- `Defekten Link in README zum White Paper beheben`
- `Prozess zur Festlegung der Basislinie klären`
- `Emotionales Vokabular Eintrag für 'anxious' erweitern`

**Vermeiden:**
- `Datei aktualisieren`
- `Korrekturen`
- `Änderungen`

### Dateibenennungskonventionen

Befolgen Sie bestehende Konventionen:
- Verwenden Sie Bindestriche für Leerzeichen
- Fügen Sie Versionsnummer in Protokolldateien ein: `(RC0001)`
- Fügen Sie Sprachversion ein: `(US-English)`
- Verwenden Sie beschreibende Namen

**Beispiel**: `TriLateral-Voice (TLV) Protocol-Name (RC0001)(US-English).md`

## Überprüfungsprozess

1. **Einreichung**: Sie reichen einen Merge Request oder ein Issue ein
2. **Erstprüfung**: Das TLV-Team prüft innerhalb von 10-15 Werktagen
3. **Diskussion**: Das Team kann Fragen stellen oder Änderungen anfordern
4. **Überarbeitung**: Sie nehmen alle angeforderten Änderungen vor
5. **Genehmigung**: Nach Genehmigung werden Änderungen zusammengeführt
6. **Anerkennung**: Mitwirkende werden anerkannt

**Zeitrahmen**: Die meisten Beiträge werden innerhalb von 2 Wochen überprüft. Komplexe Beiträge (neue Protokolle, größere Änderungen) können länger dauern.
